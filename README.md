> Self-taught programmer based in New Haven, CT, studying game design and trying to make a career out of it.

I mostly build things for myself: tools that collect and visualise data, stuff that scratches an itch I have, things that look good and actually work. If someone else ends up finding them useful too, that's a bonus.

I lean toward math-heavy problems and care a lot about the details, both how something functions and how it looks. Most of my projects start because I needed the thing to exist and it didn't yet.
